
# 眼在手上（Eye-in-Hand）手眼标定流程

## 概述

**眼在手上**配置：相机安装在机械臂末端，ArUco 标记固定在外部（如工作台）。

**标定目标**：求解 `camera_color_optical_frame → elfin_end_link` 的变换矩阵。

## 1. 准备工作

### 1.1 硬件安装
- 将相机牢固安装在机械臂末端法兰上
- 将 ArUco 标记固定在工作空间内（如桌面），确保标记不会移动
- 建议使用较大的 ArUco 标记（10cm 以上），提高检测精度

### 1.2 检查 ArUco ID
确认 ArUco 检测节点配置的标记 ID 与实际使用的标记一致。

## 2. 启动相机

在 `ros2_ws` 工作空间中启动相机：

```bash
ros2 launch orbbec_camera gemini_330_series.launch.py enable_colored_point_cloud:=true
```

## 3. 启动二维码检测

在 `robot_dev_vision` 工作空间中启动 ArUco 检测节点：

```bash
ros2 launch aruco_pose_estimation aruco_pose_estimation.launch.py
```

**验证检测**：可在 RViz 中查看 `/aruco/image` 话题确认检测效果。

## 4. 运行标定程序

在 `elfin_SDK` 工作空间中运行眼在手上标定器：

```bash
ros2 run elfin_sdk_control_py hand_eye_calibrator_eye_in_hand
```

## 5. 采集标定数据

### 5.1 操作要点
1. 移动机械臂到不同位置和姿态
2. 确保 ArUco 标记始终在相机视野内
3. 输入 `y` 采集当前数据点
4. 重复以上步骤，采集至少 **10 组** 不同姿态的数据

### 5.2 姿态要求
- **位置多样性**：机械臂应移动到不同的 X/Y/Z 位置
- **角度多样性**：末端应有不同的倾斜角度（绕 X/Y 轴旋转）
- **避免共线**：采集点不应在同一条直线上

### 5.3 交互命令
| 命令 | 功能 |
|------|------|
| `y` | 采集当前姿态数据 |
| `c` | 执行标定计算 |
| `q` | 退出程序 |

## 7. 使用标定结果

### 7.1 发布 TF

创建一个 TF 广播节点，使用标定结果发布 `elfin_end_link → camera_color_optical_frame` 的变换：

```python
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import TransformStamped
import tf2_ros

class HandEyeTFBroadcaster(Node):
    def __init__(self):
        super().__init__('hand_eye_tf_broadcaster')
        self.tf_broadcaster = tf2_ros.TransformBroadcaster(self)
        self.timer = self.create_timer(0.02, self.broadcast_tf)  # 50Hz
        
        # 从标定结果填入 (T_end2cam)
        self.translation = [0.0523, 0.0312, 0.0856]  # 根据实际结果修改
        self.quaternion = [0.0100, 0.0045, 0.7071, 0.7070]  # [x,y,z,w]
    
    def broadcast_tf(self):
        t = TransformStamped()
        t.header.stamp = self.get_clock().now().to_msg()
        t.header.frame_id = 'elfin_end_link'
        t.child_frame_id = 'camera_color_optical_frame'
        t.transform.translation.x = self.translation[0]
        t.transform.translation.y = self.translation[1]
        t.transform.translation.z = self.translation[2]
        t.transform.rotation.x = self.quaternion[0]
        t.transform.rotation.y = self.quaternion[1]
        t.transform.rotation.z = self.quaternion[2]
        t.transform.rotation.w = self.quaternion[3]
        self.tf_broadcaster.sendTransform(t)
```

### 7.2 TF 树结构

标定完成后，TF 树应为：
```
elfin_base_link
    └── elfin_link1
        └── ... (机械臂连杆)
            └── elfin_end_link
                └── camera_color_optical_frame  ← 通过标定结果发布
                    └── camera_depth_optical_frame
                    └── ...
```

## 8. 验证标定精度

### 8.1 可视化验证
1. 在 RViz 中显示 TF 树
2. 将 Fixed Frame 设置为 `elfin_base_link`
3. 检查相机坐标系是否正确跟随机械臂末端移动

### 8.2 定量验证
- **平均误差** < 5mm：标定结果良好
- **平均误差** 5-10mm：可接受，可尝试增加采集点数
- **平均误差** > 10mm：需重新标定，检查是否有以下问题：
  - ArUco 标记检测不稳定
  - 相机安装松动
  - 采集姿态不够多样

## 常见问题

### Q1: 检测不到 ArUco 标记
- 检查光照条件
- 确认标记 ID 配置正确
- 调整相机曝光参数

### Q2: 标定误差过大
- 增加采集点数到 15-20 个
- 确保姿态多样性
- 检查相机是否松动

### Q3: 如何确定偏移方向
- `T_end2cam` 用于发布 TF（父帧: end_link, 子帧: camera）
- `T_cam2end` 用于将相机坐标系中的点转换到末端坐标系
