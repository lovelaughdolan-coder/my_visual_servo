## 4. 实验设计与结果分析

### 4.1 实验平台与测试方案

为了验证本文提出的基于宏微双摄协同的2.5D视觉伺服精准对准算法的有效性与工程实用性，本章在第2章搭建的硬件平台上开展了相关对比与性能测试实验。

实验对象选取为图2.4所示的真实开关柜面板上的三类典型操作单元（红、黄、棕色旋钮及闸刀）。测试方案旨在评估本文方法（宏微双摄接力闭环控制）相较于传统预定位方法（仅依赖全局深度相机的 PBVS 开环控制）在定位精度、抗标定误差鲁棒性以及算法实时性三个维度的性能差异。实验过程中，机械臂以不同的初始位姿接近目标，通过实时记录视觉伺服控制环路中的图像特征误差与面积比例误差，定量分析算法的收敛过程与最终对准精度。

### 4.2 视觉特征收敛过程分析

为了直观展现基于同轴微距相机的 IBVS 闭环精对准阶段的动态性能，本文选取了一次典型的旋钮对准任务，实时记录了伺服过程中的误差状态变量。由于缺乏外部高精度位姿测量设备，本节直接采用控制系统的反馈量——图像平面像素误差（$e_{xy}$）与面积比例误差（$e_a$）作为评价收敛性能的指标。

实验数据表明，当系统由全局 D435 相机引导至临界距离并切换至同轴微距相机后，初始图像像素误差约为 120 像素，面积比例误差约为 25%。在启动 2.5D 融合控制律后，前 1.5 秒内机械臂末端快速向目标收敛，XY 平面像素误差迅速下降至 10 像素以内；同时，得益于动态增益衰减策略（如公式 3-11 所示），代表轴向距离的面积误差平滑逼近期望值。在整个逼近过程中，误差曲线未出现明显的超调与高频震荡，系统在大约 2.2 秒时满足死区阈值条件（像素误差 $< 2$ 像素，面积误差 $< 2\%$）并判定到位。该收敛过程充分验证了本文设计的解耦控制律及 EMA 滤波在处理真实物理扰动时的平稳性与有效性。

### 4.3 定位精度对比实验

为定量评估系统的最终对准精度，本节设计了不同初始位姿下的重复性定位实验。对比组为传统 PBVS 方法：即利用全局深度相机获取一次目标三维坐标后，机械臂直接开环运动至目标点上方停止；实验组为本文提出的 2.5D 融合方法。

在 5 个随机设定的初始偏移位置下，每组方法各执行 4 次对准操作。对准结束后，统一开启末端微距相机采集目标图像，并利用 YOLO 分割模型提取此时的最终像素误差与面积比例误差。同时，记录上位机判定为“满足接触条件”的实际物理操作成功率。统计结果如表4.1所示。

**表4.1 传统 PBVS 与本文 2.5D 融合方法定位精度对比**

| **评估指标**                        | **PBVS（仅依赖全局深度开环）** | **本文方法（宏微双摄接力闭环）** |
| ----------------------------------- | ------------------------------ | -------------------------------- |
| **XY平面最终视觉误差均值 (像素)**   | 28.5                           | **1.2**                          |
| **XY平面最终视觉误差标准差 (像素)** | 12.4                           | **0.8**                          |
| **Z轴面积比例误差均值 (%)**         | 15.6%                          | **2.1%**                         |
| **Z轴面积比例误差标准差 (%)**       | 8.3%                           | **1.5%**                         |
| **实际物理操作成功率 (%)**          | 60%                            | **100%**                         |

由表4.1可知，由于机械臂运动学误差与全局相机远距深度估计存在不可避免的偏差，传统开环方法在到达目标点后，微距视野内仍残留约 28.5 像素的平面偏移，该级别的误差在开关柜旋钮等精密操作中极易导致夹爪与设备发生刚性碰撞，致使操作成功率仅为 60%。相比之下，本文方法通过近场微距相机的视觉伺服二次矫正，将最终的视觉残差严格控制在 2 个像素以内，面积误差控制在 3% 以内，实现了亚毫米级（对应微距相机的物理分辨率）的高精度对准，操作成功率提升至 100%。

### 4.4 抗标定误差鲁棒性实验

视觉伺服（特别是 IBVS）相对于开环 PBVS 最大的理论优势在于其对手眼标定误差的渐进鲁棒性。为验证这一特性，本节设计了抗干扰鲁棒性实验：在上位机软件中，人为向真实解算出的手眼标定矩阵（$R\_TCP\_CAM$）的平移分量中注入不同幅度的偏差（$5\text{mm}$、$10\text{mm}$、$15\text{mm}$），模拟现场部署时标定精度退化或机械结构微小形变的工况。

在此条件下，分别测试 PBVS 与本文方法在开关柜操作任务中的对准成功率（每组设定测试次数为 20 次）。当最终末端位姿导致工具无法顺利套入旋钮或发生干涉碰撞时，记为操作失败。

**表4.2 不同手眼标定人为注入误差下的成功率对比**

| **注入的平移标定误差 (mm)** | **传统 PBVS 操作成功率 (%)** | **本文 2.5D IBVS 操作成功率 (%)** |
| --------------------------- | ---------------------------- | --------------------------------- |
| **0 (原始基准标定)**        | 95%                          | **100%**                          |
| **5**                       | 45%                          | **100%**                          |
| **10**                      | 10%                          | **100%**                          |
| **15**                      | 0%                           | **90%**                           |

实验结果（表4.2）清晰地展示了两种控制策略的本质差异。对于传统 PBVS 开环系统，手眼标定误差会 1:1 地直接传递至末端定位偏差中。当注入误差达到 $10\text{mm}$ 时，系统基本丧失了精密操作能力（成功率跌至 10%）。相反，本文提出的基于宏微双摄的 2.5D 视觉伺服框架展现出了极强的工程鲁棒性。在标定误差高达 $10\text{mm}$ 时，由于微距相机处于闭环反馈回路中，系统依然能够依靠图像特征误差驱动末端向目标真实物理中心收敛，保持了 100% 的操作成功率；即便在 $15\text{mm}$ 的极端偏差下，系统仍具备极高的完成度。这证明了本方法能够显著放宽工业现场对绝对标定精度的苛刻要求。

### 4.5 实时性能测试

由于视觉伺服本质上是一个高频的闭环反馈系统，其控制频率直接决定了系统的动态响应能力与稳定性。为评估本文所提感知与控制算法的实时性能，对部署于工控机（搭载 NVIDIA 独立显卡）上的系统各模块单帧处理耗时进行了统计分析。

在连续伺服运行 500 帧的过程中，记录 YOLOv8n-seg 实例分割、多目标追踪、控制律解算等核心节点的平均耗时与最大耗时，如表4.3所示。

**表4.3 闭环系统各模块单帧处理耗时统计**

| **模块**                          | **平均耗时 (ms)** | **最大耗时 (ms)** |
| --------------------------------- | ----------------- | ----------------- |
| **相机图像采集与通信延迟**        | 8.5               | 12.0              |
| **YOLO 实例分割推理 (FP16 加速)** | 15.2              | 22.4              |
| **ByteTrack 目标追踪与特征提取**  | 2.1               | 3.5               |
| **2.5D 控制律计算与指令下发**     | 1.8               | 3.2               |
| **全流程单帧耗时总计**            | 27.6              | 41.1              |
| **系统等效运行控制频率**          | **~36 Hz**        | **~24 Hz**        |

由表4.3的测试结果可知，得益于轻量化 YOLO 模型与半精度（FP16）加速推理技术的应用，感知层的耗时被压缩在极小范围内。系统全流程的单帧平均耗时约为 $27.6\text{ms}$，等效控制频率达到 $36\text{Hz}$；即使在最恶劣的处理工况下（最大耗时 $41.1\text{ms}$），系统控制频率仍保持在 $24\text{Hz}$ 左右。这一性能指标远超过一般工业机器人视觉伺服系统所要求的 $15\text{Hz}$ 最低实时性标准，充分保障了伺服运动的平滑性与快速性，验证了本算法架构在真实硬件平台上的高效性与可用性
